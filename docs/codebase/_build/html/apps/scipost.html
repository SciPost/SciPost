scipost.views.register(request)[source]¶

This public registration view shows and processes the form that will create new user account requests. After registration the Contributor will need to activate its account via the mail sent. After activation the user needs to be vetted by the SciPost admin.

scipost.views.unsubscribe(request, contributor_id, key)[source]¶

The link to this method is included in all email communications with a Contributor. The key used is the original activation key. At this link, the Contributor can confirm that he/she does not want to receive any non-essential email notifications from SciPost.
